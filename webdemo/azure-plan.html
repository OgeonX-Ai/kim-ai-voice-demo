Upgrade path

Swap Whisper for Azure Speech

Keep the same browser UI and gateway, but point /v1/audio/transcribe-file to Azure for faster, scalable STT.

Architecture

Browser → enterprise-ai-gateway → Azure Speech → text response. API keys live in Key Vault, never in the browser.

  • Use the same POST /v1/audio/transcribe-file endpoint
  • Configure Azure Speech credentials server-side (Key Vault recommended)
  • Return text + latency metrics for the UI cards

Setup steps

  1. Provision Azure Speech — create resource and grab the endpoint/key.
  2. Store secrets — add the key to Azure Key Vault and wire the gateway to fetch it at startup.
  3. Update gateway route — forward uploaded audio to Azure Speech, keep the response shape for the UI.
  4. Test locally — hit /healthz and /v1/audio/transcribe-file from the Whisper Call section.
  5. Deploy — run the gateway where your users are; the UI stays on GitHub Pages.

Why Azure?

  • Lower latency for Finnish and English
  • Scales horizontally; no need to ship GPU boxes
  • Enterprise security: secrets in Key Vault, auth controls in the gateway
